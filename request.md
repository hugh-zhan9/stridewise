# StrideWise / 睿步 需求文档（request.md）

## 1. 文档目标
本文件用于定义一款“基于天气与训练数据的智能运动建议产品”的可执行需求。
目标是让 AI、产品、研发、测试在同一文档下对功能边界、规则和验收标准达成一致。

## 2. 产品目标与成功标准
### 2.1 产品目标
- 为不同能力层级用户提供个性化、可解释、可执行的每日训练建议。
- 在保证安全前提下提升训练连续性与完成率。
- 通过训练后反馈与表现数据持续更新用户能力基线。

### 2.2 核心成功指标（MVP）
- 建议采纳率 >= 35%
- 建议后训练完成率 >= 70%
- 7日留存率 >= 25%
- 训练后“不适反馈率” <= 5%

## 3. 目标用户与场景
### 3.1 目标用户
- 新手跑者：缺乏训练计划，需要低风险、易执行建议。
- 进阶跑者：已有基础，希望结合天气和身体状态优化训练。

### 3.2 关键场景
- 用户每日打开应用，获取“今天是否适合跑”和“跑什么”。
- 用户记录/导入训练数据后，获得训练总结和后续建议调整。
- 天气恶劣或疲劳高时，系统自动降级建议或建议休息。

## 4. 功能范围
### 4.1 P0（MVP 必须）
1. 用户基础信息与目标设置
2. 能力基线评估与动态更新
3. 天气数据接入与风险分级
4. 训练记录（手动）
5. 第三方数据导入（首期仅 1 个渠道）
6. 每日训练建议（AI 优先 + 规则兜底）
7. 训练后总结与“建议是否有用”反馈
8. 安全保护与强制降级策略
9. 异常兜底流程

### 4.2 P1（迭代）
1. 多数据源扩展（如 Strava + Garmin + Nike + GPX/TCX/FIT）
2. 个性化参数自动学习
3. 周/月训练趋势分析

### 4.3 P2（长期）
1. 建议策略由“AI 优先 + 规则兜底”向更强模型化能力演进
2. 更细粒度疲劳与恢复评分

## 5. 详细功能需求
### 5.1 用户信息与目标管理
- 输入字段：性别、年龄、身高、体重、静息心率（可选）、运动目标（减脂/健康维持/5K提升等）、位置（经纬度 + 国家/省/市，必填）。
- 能力层级：新手/中级/进阶（由历史训练数据自动判断，支持手动调整）。
- 支持长远目标设置：目标周期、目标频次、目标配速或完赛目标。

验收标准：
- 首次进入可在 3 分钟内完成建档。
- 无历史数据用户可通过问卷初始分层并正常获取建议。

### 5.2 能力基线评估
- 输入：最近 28 天训练记录（最少 7 天可触发基础评估）。
- 输出：能力基线（建议配速区间、建议周训练量区间、恢复需求等级）。
- 更新时机：每次训练结束后 + 每日夜间批处理。

验收标准：
- 每次训练后 1 分钟内完成基线更新。
- 用户可查看“基线变化原因”摘要（如负荷上升、恢复不足）。

### 5.3 天气数据接入与风险分级
- 天气必需字段：温度、体感温度、湿度、风速、降水概率、AQI、紫外线（依赖用户位置，位置必填）。
- 风险分级：
  - 绿色：适宜户外训练
  - 黄色：可训练但需降强度/缩时长
  - 红色：不建议户外跑，触发替代方案

验收标准：
- 每日建议生成前必须完成天气拉取。
- 天气缺失时触发“保守建议”并提示数据异常。

### 5.4 训练记录与数据导入
- 手动最小字段：训练类型、开始时间（精确到时间）、时长、距离、平均配速、RPE（1-10）、是否不适。
- 第三方导入支持来源范围：Keep、Strava、Garmin、Nike、GPX、TCX、FIT（按迭代优先级分阶段接入，P0 首发 Keep）。
- 数据冲突策略：第三方数据优先；若手动记录与第三方记录时间区间重叠，拒绝手动记录（不允许覆盖）。

验收标准：
- 手动记录一次训练耗时 <= 60 秒。
- 导入失败可重试并给出可理解错误原因。

### 5.5 每日训练建议引擎（AI 优先 + 规则兜底）
- 决策输入：
  - 近 7 天训练负荷
  - 当日与未来 3 天天气窗口
  - 当前能力基线
  - 恢复状态（含睡眠或身体电量，如有）
- 决策输出（结构化）：
  - 今日是否建议跑
  - 推荐类型（恢复跑/耐力跑/间歇/休息）
  - 建议强度区间
  - 建议时长/距离
  - 推荐时段
  - 风险提示与原因解释
- 规则兜底：
  - AI 建议生成后必须经过规则校验
  - 红色风险触发强制降级或替代方案

验收标准：
- 建议必须可解释，显示至少 2 条依据。
- 新手与进阶用户在同天气条件下建议结果应可区分。

### 5.6 天气驱动的替代运动建议
- 当天气或风险不适合户外跑时，给出替代方案：室内跑步机、力量训练、拉伸恢复。
- 同时给出补水提醒、穿衣建议、训练前后注意事项。

验收标准：
- 红色风险下不得输出“户外高强度跑”建议。

### 5.7 训练后总结与反馈闭环
- 训练后自动生成总结：完成度、强度匹配度、恢复建议。
- 用户反馈入口：建议是否有用（有用/一般/无用）+ 可选原因。
- 反馈用于后续个性化策略微调。

验收标准：
- 总结在训练结束后 30 秒内可见。
- 反馈提交成功率 >= 99%。

### 5.8 安全保护模块
- 触发条件（任一满足即触发保护）：
  - 天气红色风险
  - 连续高负荷且恢复不足
  - 用户主动上报不适
- 保护行为：
  - 强制降级建议或建议休息
  - 给出风险说明，不允许继续推荐高强度训练

验收标准：
- 安全规则优先级高于所有训练提升目标。

### 5.9 异常场景与兜底
- 新用户无历史数据：使用问卷+保守建议模板。
- 天气 API 失败：使用最近一次有效天气 + 保守建议。
- 第三方导入失败：允许手动补录并提示重试。
- 关键字段缺失：降级生成建议并标记可信度低。

验收标准：
- 以上异常场景都必须可完成建议输出，不可中断主流程。

## 6. 规则兜底优先级
规则优先级从高到低：
1. 安全规则
2. 恢复状态规则
3. 天气风险规则
4. 训练目标规则
5. 用户偏好规则

冲突处理原则：高优先级规则可覆盖低优先级输出。

## 7. 数据模型（MVP）
### 7.1 用户档案 UserProfile
- user_id
- 基础信息（性别、年龄、身高、体重）
- 运动目标
- 能力层级
- 位置（经纬度 + 国家/省/市，必填）
- 创建时间/更新时间

### 7.2 训练记录 TrainingLog
- log_id, user_id
- 日期、训练类型、时长、距离、配速、RPE
- 不适标记
- 数据来源（manual/third_party）

### 7.3 天气快照 WeatherSnapshot
- snapshot_id, user_id, 日期
- 温度、湿度、风速、降水概率、AQI、紫外线、风险等级

### 7.4 建议记录 Recommendation
- rec_id, user_id, 日期
- 建议内容（结构化字段）
- 生成依据摘要
- 风险等级

### 7.5 反馈 Feedback
- feedback_id, rec_id, user_id
- 是否有用
- 反馈文本（可选）
- 提交时间

## 8. 核心流程（闭环）
1. 用户初始化档案与目标
2. 拉取天气 + 读取近7天训练 + 获取能力基线
3. 规则引擎生成当日建议
4. 用户执行训练并记录/导入
5. 系统输出训练总结
6. 用户提交“建议是否有用”
7. 系统更新能力基线并影响下一次建议

## 9. 非功能与合规要求
- 隐私：明确告知数据用途，第三方授权需可撤销。
- 可靠性：建议生成链路可用性 >= 99.5%。
- 可观测性：记录建议输入、输出、命中规则、异常原因。
- 合规文案：本产品为运动建议工具，不替代医疗建议。

## 10. 上架策略
### 10.1 内测上架条件（必须全部满足）
1. P0 功能全部完成
2. 安全保护规则全量生效
3. 异常兜底场景验证通过
4. 核心指标可采集并可看板化

### 10.2 公开上架前补充要求
1. 完成至少 2 周灰度数据验证
2. 不适反馈率稳定低于目标阈值
3. 数据授权与隐私流程通过法务评估

## 11. 待确认项（产品决策）
1. 第三方首发数据源选型：已确认 P0 首发 Keep（后续依次扩展 Strava、Garmin、Nike、GPX/TCX/FIT）
2. “身体电量”计算方式：仅基于主观反馈，还是接入睡眠/心率数据
3. MVP 是否包含周报功能
